System requirements

System requirements

  • Windows 10 64 Bit or Windows 11 64 Bit
  • MySQL or MariaDB Database-server (no server needed by SQLite)
  • ODBC Database Driver (see Database)
  • Dark Mode enabled

For compiling you need:

  • Lazarus 4.0 and FreePascal Compiler (FPC) 3.2.2
  • MetaDarkStyledsgn Package
  • ATSynEdit Package
  • RichMemo Package
  • PowerPDF Package
  • UniqueInstance Package

Database

Database

After first launch you need to install / setup the datatbase.
Direct on startpage or click the symbol shown to the left in the top menu.

For the database connection you need a 64Bit ODBC-Driver equal to the server.

MySQL:

ODBC-Driver Website

MariaDB:

ODBC-Driver Website

SQLite:
or SQLite you dont need a server running, but you need a ODBC-Driver.

ODBC-Driver Website

You can enter the database login by ODBC-DSN or enter them in the database dialog.
If you do not use ODBC-DSN, enter the login data in the equal fields.
On Linux you must use DSN.

After click on test connection, click on use. the given data will be saved and used every time the program starts.
You can change the settings every time you like.
On positiv connection click on install / restore. You will be asked to use Schnipsel.sql. click on yes.
The Tables in the database will be created.
Please note that everytime you use that function, tables will be erased / overwritten

Backup:
 You can build a backup sql-file of the database, click on backup and follow the dialogs.

Restore:
 Here you be able to reset the database or restore the database by a backup file.
When pushing that button you will be asked to use Schnipsel.sql. Click on no if you like to choose a SQL-file.

Add / edit Code-Language / Menuentry

Add / edit Code-Language / Menuentry

To add a new Code-Language, click on the left displayed symbol in the top menu. (last icon),
and there choose edit categories and languages.

You get a new dialog.
Enter a new Language, and a short term that will be displayed in the menu.Example: 'Pascal' and 'Pas'.
Click on save.

To edit a coding-language, click the same things as above.

In opening dialog.
Select the entry you like to change under edit. the fileds below the list will be filled with the current values. Edit as you like.
Click on update.

To delete a coding language, click the same things as above.

In opening dialog.
Select the entry you like to delete under delete.
In the list below you can select a coding-language where the code-entrys of the deleted coding-language will be moved. Choose the same entry as
above to delete also all code-entrys.
Click on delete

Add / edit Code-Type

Add / edit Code-Type

To add a new Code-Type, click on the symbol shown to the left in the top-menu,
and then on Languages->Types.

You get a new Dialog.
Select a Language in the list.Enter a new Type. Example: 'Function' or 'Procedure'.
Click on save.

To edit a Code-Type, click the same as above, and then Language - Types.

In the open Dialog,
Select the type you like to edit in the section Edit. The Type will be shown in the edit field, give it a new name and click on update.

To delete a Code-Type, click the same as above.

In the open Dialog,
Select the entry you like to delete. In the list below you can select the type where all code-entrys of the deleted type will be moved to.
If you select the same as the deleted type, all code-entrys within this type will be also deleted.
Click on delete.

Add Code-Entrys

You can create multiple versions of a code-entry.
If you like to add a new version have a look at New Code Version.

Click on the left symbol in the top menu.


In the open Dialog select a language and type, and enter the name for the code-entry.
Click on save.

The Edit-Mode will be active and the editor is shown. Enter version, author and insert the code.

Click on save (disk-symbol).

Add new Code-Version

Activate Edit-Mode by clicking on the left symbol in the top-menu.

Go to the code you like to add a new version.
Above the code there are shown some symbols.

Click on the left symbol.

The editor will be shown.
Enter version, author and add the code.
Click on save (the disk-symbol).

Above the code there will be a new button shown.
By click on these V-buttons you can change between the code-versions.

Edit Code-Entrys and Code-Versions

Activate the edit-mode with click on the left symbol in the top-menu

Go to the code you like to edit.
Above the code there will be some symbols.



Select the version you like to edit with the V-Buttons.

Click on the left symbol above the code. The code will be displayed in the editor.
Edit the code as you like and click on save (the disk-symbol).

Delete Code-entrys

Activate the edit-modus by clicking on the left symbol in the top-menu.

Go to the code-entry you like to delete.

Click on the left symbol above the code to delete the code-entry and all the versions of it.


Only delete one Version of a entry

Delete a Code-Version

Activate the edit-modus with clicking on the left symbol in the top-menu.
Go to the Code-entry you like to delete a version of.

Click on the left symbol above the code to delete it.


Delete a code and all versions of it

Export Codeentrys

Activate the export-mode by clicking on the left symbol.

Go to the code-entry you like to export.
Above the code there will be the left symbol displayed, click on it. In the open dialog you can select
wich parts of the code-entry will be exported. The code to export will be displayed in the export-list as a bigger graphic.
You can export as many code-entrys as you like.

Move the mouse over a graphic in the exportlist will show the name of the code.
To delete a code from the exportlist you only have to click on it.

Click on a button below the exportlist, deppending on the format you like to export in.
You can choose a path and filename for the export.

If you export into html, the selected options of the settings-dialog will be used.

Export to HTML

Export to HTML let you choose colors and templates in the settings-dialog.
The templates are located in the subdirectory Templates and you can edit and create them.
There are some things to know about:

  • Files:
    In Templates you create CSS-files. The HTML-part of the exported file is allways the same and canÂ´t be edited,
    see below.
    You can name the CSS-file as you like, you only got to add "List_" to the name if you like to use clickable list in
    the export-file. Sample-graphics of the templates can be displayed. Create them with the same name as the CSS-file under Templates/Images
    in the png format. Please use a size near 600x300 Pixels.
    A Dummy-graphic is displayed if there is no similar file


  • Colors:
    For colors, if you dont use your own in the CSS-file, you can use the preselected colors in the settings-dialog with variable-names.
    [COLOR1], [COLOR2], [COLOR3], [COLOR4], [COLOR5] for the preselected color-theme,
    [TXTCOLOR] for the preselected Textcolor.

  • HTML:
     The HTML-part of the export is allways the same and can only be styled with the CSS-part. This doesn`t affect on Header.html
    and Footer.html.


    <!DOCTYPE html>
